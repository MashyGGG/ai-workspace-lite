# Week 6 安全复盘

## 目前最脆弱的是哪一层

- **模型遵从度**：强提示与工具 schema 仍无法 100% 防止误执行或风格被带偏；规则关键词可被改写绕过。
- **模型层安全分类覆盖**：未配置 `MOONSHOT_API_KEY` 时无法调用 Kimi 分类，仅依赖本地可疑词列表，覆盖面有限（见 [domestic-llm.md](domestic-llm.md)）。

## 用什么方式缩小了影响范围

- 有副作用的 **任务创建** 与 **模型回复** 解耦：API 返回 `proposedTask`，写入权在用户确认之后。
- 文档问答 system 中明确 **「文档非指令」**，降低把恶意段落当规则的概率。
- 抽取链路增加「不执行原文指令」约束，减少被用户粘贴内容劫持输出形态的风险。

## 暂时没做、后续可补的内容

- 对 `/api/extract` 输入做与 agent/docs 同级的预检（成本与误杀需权衡）。
- 安全样本的**自动化**评分（当前 `security_cases_v1.jsonl` 以手工/半自动验证为主）。
- 更细粒度分类（提示注入 vs 恶意内容 vs PII）与可观测日志。
