# Week 2 — 失败案例复盘（周日）

## 案例：极短、无责任人与截止的「口号式」任务描述

**输入特征：** 只有「尽快推进」「大家分头落实」等表述，没有具体事项、负责人或时间。

**现象：**

- `actionItems` 有时会被模型勉强拆成 1～2 条泛化任务（如「推进项目」），与原文可执行性不匹配，**抽取不稳定**。
- `risks` 里若原文没有明确威胁，`severity`（低/中/高）容易依赖模型主观判断，**等级偏主观**、不同次调用可能不一致。

**下周可改进方向：**

- 在 system 或 schema 说明中强调：无明确执行主体与时点时，`actionItems` 应返回空数组，把不确定性放进 `openQuestions`。
- 为 `severity` 增加更可操作的分级规则（例如：仅当原文出现「延期/未确认/依赖外部」等信号时才标 `medium` 以上），并在评测里对同输入多次调用看方差。
